Lesson 1: The Basics Of The Web and HTML

The World Wid Web

The web uses a computer language called HTML which is short for Hypertext Markup Language. The main type of document that's on the web is actually HTML, so we know that this is important. There are many different file types within the web; these include plain text, HTML, images, video, and music. There are also some major pieces of the web: you, the browser, internet, and the servers. HTTP is the protocol of the web. The browser is the program that runs to display files on the web. Internet is the computer network. Servers are the computers that do the hosting of the files. If you'd like more information, then you can visit this link.
Summary - There are many different pieces of the web including the internet , servers, HTTP and the browser. HTML communicates with these as well as all different file types.

HTML Basics

HTML is made up of text content, which is what you see. It also contains markup, which is what it looks like. There are also references to other documents, which can be, for example, images and video. HTML markup is made up of tags, and these are incredibly important.
Summary - HTML is the structure of the web.

Tags and Elements

The way tags are structured is they start with a less than symbal (<) followed by the tag name, a greater than symbol (>), the tag contents (those can also be other tags) or it could be text, another less than symbol (<), a slash (/), the name, and finally another greater than symbol (>). So there is essentially an opening tag, the contents, and a closing tag. The whole thing combined is called an element. I won't go over how to use the bold and emphasis tags because I'm using them in these notes. There's also the anchor element which allows you to input links (hyperlinks) easily. It is important to keep in mind that computers are very literal, so always be looking for typos and small mistakes. For example, if you leave an emphasis tag open and forget to close it, then everything after the emphasis tag will be emphasized instead of just the words or phrases that you want.
Summary - The whole tag is called an element, and there's a wide range of tags that you can use.

Void Tag, Inline, and Block Elements (and what they mean)

Up next we have the image tag. This looks like this: . The words "text" appear because I didn't input the url of the image. Keep in mind that images do not have a closing tag. This is called a void tag. A void tag is a tag that doesn't have content, so it does not need a closing tag. Keep in mind that images appear in line with text. When you use the break tag that is called inline. This just means that the line ends, and the next character wraps around to the next line. The paragraph tag works differently. This is a block element. The paragraph tag essentially makes an "invisible" box. This "invisible" box can have height and width, where inline elements are JUST text. There's also the span and div elements. The only difference between them is that span is inline, and div is block. Those two elements don't do anything other than just contain their content. A div element looks something like this: <div class="North Korea">America</div>. This creates an invisible box. Span does the same thing if you use a "class" except it doesn't create an invisible box. Both of these elements have an attribute called classes in this case. . Classes are names that you can give to an element.
Summary - A void tag yields no content, so it doesn't need a closing tag. Inline tags are just lines, so when the line ends the next character wraps around. Block tags/elements create an invisible box around the content that has height and width, whereas inline elements are just text. Div element is block and span element is inline. Those two elements only contain their content. Those two elements also have an attribute called classes and those contain whatever you put inside of them.

Classes, Document Structure

Classes are important in CSS. CSS is a separate language that adds style to your documents/webpages. Up next we have HTML Document Structure . First there's the DOCTYPE. The DOCTYPE is HTML. Next we have the opening HTML tag and the closing HTML tag. This surrounds the rest of the document. Next there's the head. What this contains is metadata and other random things. If we included CSS, Javascript, etc that would be included in the head. There's also the title. The title is the title of the page which appears at the top of the browser window/tab.
